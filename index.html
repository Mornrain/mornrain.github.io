nodemon

(GitHub Alias)

nodemon is a tool that helps develop Node.js based applications by automatically restarting the node application when file changes in the directory are detected.

Installation

npm install --save-dev nodemon
# or
yarn add nodemon -D

Usage

将 nodemon 定义为开发依赖 development dependency 后，package.json 的内容改变如下:

{
  //...
  "dependencies": {
    "express": "^4.17.1",
  },
  "devDependencies": {
    "nodemon": "^2.0.7"
  }
}

最好是在 package.json 文件中为它定义一个专用的 npm 脚本:

{
  // ..
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js",    "test": "echo \"Error: no test specified\" && exit 1"
  },
  // ..
}

在脚本中，不需要指定 node_modules/.bin/nodemon 到 nodemon ，因为 npm 自己知道从该目录搜索文件。

我们现在可以在开发模式下使用如下命令启动服务器:

npm run dev

与 start 和 test 脚本不同，我们还必须将 run 添加到命令中。

在 Create React App 中使用 Typescript

项目配置

Create React App 内置了对 TypeScript 的支持。

需要创建一个使用 TypeScript 的新项目，在终端运行：

npx create-react-app my-app --template typescript

下载 React Router：

npm install react-router-dom@6

使用上下文

此钩子用于获取 Provider 的当前上下文。为了创建和提供上下文，我们使用 React.createContextAPI。

interface 与 type 区别

  • Interfaces vs. Type Aliases
  • Interface vs Type alias in TypeScript 2.7
  • Types vs. interfaces in TypeScript

useEffect

  • 会在每次 render 之后执行
  • 接受第二个参数来控制跳过执行，下次 render 后如果指定的值没有变化就不会执行
  • useEffect 是在 render 之后，也就是浏览器已经渲染结束才执行

零碎知识点

  • React 项目会以 public 文件夹为根目录

复制

const myContext = React.createContext()

我们将根组件放在 myContextProvider 之间：

function App() { 
    return ( 
        <myContext.Provider value={900}> 
            <Root /> 
        </myContext.Provider> 
    ) 
}

为了消耗由我们提供的值，我们使用 useContext 钩子。

function Root() { 
    const value = useContext(myContext) 
    return ( 
        <div> 
            <h3>My Context value: {value} </h3> 
        </> 
    ) 
}
