在 Create React App 中使用 Typescript

项目配置

Create React App 内置了对 TypeScript 的支持。

需要创建一个使用 TypeScript 的新项目，在终端运行：

npx create-react-app my-app --template typescript

下载 React Router：

npm install react-router-dom@6

useEffect

  • 会在每次 render 之后执行
  • 接受第二个参数来控制跳过执行，下次 render 后如果指定的值没有变化就不会执行
  • useEffect 是在 render 之后，也就是浏览器已经渲染结束才执行
  • However, an effect hook should return nothing or a clean up function.

问题记录

1. react 的 spa 应用锚点跳转的问题

场景

在 Bar 页面内有一个自己页面内的锚点，点击后 url 发生了改变，锚点跳转失效。

原因

猜测？（使用了 link 组件，在 Bar 页面内点击新的锚点，Link 组件使用 push 或 replace 产生了新的 window.history 记录，这个新的 window.history 的 pathname 和以前一样，只是有新的 hash。然后就去 Route 里去找匹配组件 Bar 组件。因为已经渲染过了，就只触发重渲染。而此时的 protocol，port，pathname 都没变，只是 hash 变了，浏览器认为这是老页面上的一个新锚点，就在老页面上直接跳转。这时存在的问题是页面组件在重渲染，而此时在浏览器刚要跳转到新锚点的时候重渲染导致锚点没了，跳转失败。）
